系統架構

現代化技術棧與彈性架構設計

架構概述

微服務導向的多層次架構

功德辦理系統採用現代化的微服務架構，結合先進的前後端分離設計，實現高可擴展性、高可用性和高安全性。 系統架構分為七大層次，從前端應用到基礎設施，形成完整的技術生態系統。特別針對雙分院業務需求， 設計獨特的資料隔離與共享機制，同時兼顧性能與維護性。

七層架構設計

前端應用層

基於 Nuxt.js 與 Vue.js 的前端框架，實現 SPA 與 SSR 混合渲染

API 閘道層

API 路由、請求轉發、安全控制、快取管理與負載均衡

應用服務層

業務微服務模組、事件驅動架構、領域驅動設計

數據存取層

GraphQL API、數據訪問物件、ORM 映射、連接池管理

數據持久層

PostgreSQL 關係型數據庫、Redis 快取、分佈式事務管理

基礎服務層

身份驗證、授權、日誌、監控、通知、排程任務、檔案存儲

基礎設施層

Docker 容器化、Kubernetes 編排、雲端部署、自動擴縮容

系統核心設計

創新架構解決方案，滿足寺院特殊需求

🏯

雙分院架構設計

針對本山與台中分院的雙分院運作模式，設計特殊的資料隔離與共享機制。系統採用多租戶架構 (Multi-tenancy) 搭配共享數據庫設計，實現業務數據隔離與基礎數據共享。每個分院擁有獨立的資料存取權限， 同時又能共享蓮友基本資料、法會範本等公共資源。

🧩

微服務架構設計

系統採用微服務架構，將各功能模組拆分為獨立的服務，各自負責特定的業務領域。每個微服務可以獨立開發、 部署和擴展，大幅提高系統的可維護性和擴展性。服務間通過 API 閘道進行通訊，並使用事件驅動模式實現鬆耦合。 微服務架構使系統能夠根據業務需求靈活調整和擴展，同時確保高可用性。

🗃️

資料庫設計

系統資料庫採用 PostgreSQL 關係型資料庫，具有強大的事務處理能力和資料完整性保障。為支援雙分院業務模式， 資料庫採用租戶識別欄位 (Tenant ID) 設計，物理共享但邏輯隔離。採用 Schema 級別的資料組織， 實現跨分院資料安全隔離。同時，系統使用 Redis 進行快取管理，提升查詢效能，並通過 MinIO 對象存儲管理文件資源。

系統部署架構

容器化部署，高可用性架構

系統採用容器化部署架構，利用 Docker 進行應用程式封裝，Kubernetes 實現容器編排與管理。 整個系統可部署於公有雲平台（如 AWS、Azure、GCP）或私有雲環境，根據需求進行彈性擴展。 系統架構支援高可用性 (HA) 設計，包含負載均衡、自動擴縮容、健康檢查與故障轉移等機制， 確保系統在各種情況下都能穩定運行。

⚖️

NGINX 負載均衡器，實現請求分發與流量控制

📈

自動擴縮容

根據系統負載動態調整資源配置，提升資源利用率

🔄

故障轉移

自動故障偵測與轉移機制，確保服務連續性

📊

監控與警報

Prometheus + Grafana 監控系統，實時監測系統狀態

系統安全性

多層次安全防護機制

🔒

身份驗證與授權

採用 OAuth 2.0 + JWT 身份驗證機制，支援多因素認證 (MFA)。 基於 RBAC (角色基礎存取控制) 的權限管理，實現精細化的資源訪問控制。 支援 API 請求簽名與時效性控制，防止請求被竄改或重放攻擊。

🛡️

資料安全

採用 TLS/SSL 加密所有傳輸中的資料，確保通訊安全。 敏感資料（如密碼）使用 bcrypt 等強加密算法存儲。 定期資料備份與異地備援，實現災難復原能力。 資料訪問審計記錄，追蹤敏感操作。

🔍

安全監控與防護

部署 WAF (Web 應用防火牆) 防止常見 Web 攻擊，如 SQL 注入、XSS 等。 實時安全監控與異常檢測，及時發現可疑行為。 定期安全漏洞掃描與滲透測試，主動發現並修復安全隱患。 DDoS 防護機制，確保系統穩定運行。
